# 🗄️ هيكلية قاعدة البيانات - نظام AZRAR

## 📋 الجداول الرئيسية

### 1️⃣ جدول الأشخاص (الأشخاص_tbl)

**الغرض:** تخزين معلومات الأشخاص (ملاك، مستأجرين، كفلاء)

| الحقل | النوع | مطلوب | فريد | الوصف |
|------|------|------|------|-------|
| رقم_الشخص | string | ✅ | ✅ PK | المفتاح الأساسي (P-timestamp) |
| الاسم | string | ✅ | ❌ | الاسم الكامل |
| الرقم_الوطني | string | ❌ | ✅ UK | الرقم الوطني (10 أرقام) |
| رقم_الهاتف | string | ✅ | ❌ | رقم الهاتف (07XXXXXXXX) |
| العنوان | string | ❌ | ❌ | العنوان |
| ملاحظات | string | ❌ | ❌ | ملاحظات إضافية |
| تصنيف | string | ❌ | ❌ | تصنيف الشخص |
| تقييم | number | ❌ | ❌ | التقييم (1-5) |

**الفهارس:**
- `PK_PersonID`: المفتاح الأساسي على `رقم_الشخص`
- `IX_NationalID`: فهرس فريد على `الرقم_الوطني`
- `IX_PhoneNumber`: فهرس على `رقم_الهاتف`

---

### 2️⃣ جدول أدوار الأشخاص (شخص_دور_tbl)

**الغرض:** ربط الأشخاص بأدوارهم (مالك، مستأجر، كفيل، وسيط)

| الحقل | النوع | مطلوب | فريد | الوصف |
|------|------|------|------|-------|
| رقم_الشخص | string | ✅ | ❌ FK | مرجع لجدول الأشخاص |
| الدور | string | ✅ | ❌ | الدور (مالك/مستأجر/كفيل/وسيط) |

**العلاقات:**
- `رقم_الشخص` → `الأشخاص_tbl.رقم_الشخص` (Many-to-One)

**ملاحظات:**
- الشخص الواحد يمكن أن يكون له أكثر من دور
- عند حذف شخص، يتم حذف جميع أدواره تلقائياً

---

### 3️⃣ جدول العقارات (العقارات_tbl)

**الغرض:** تخزين معلومات العقارات

| الحقل | النوع | مطلوب | فريد | الوصف |
|------|------|------|------|-------|
| رقم_العقار | string | ✅ | ✅ PK | المفتاح الأساسي (PROP-timestamp) |
| الكود_الداخلي | string | ✅ | ✅ UK | الكود الداخلي للعقار |
| رقم_المالك | string | ✅ | ❌ FK | مرجع لجدول الأشخاص |
| النوع | string | ✅ | ❌ | نوع العقار (شقة/محل/فيلا...) |
| العنوان | string | ✅ | ❌ | العنوان الكامل |
| المدينة | string | ❌ | ❌ | المدينة |
| المنطقة | string | ❌ | ❌ | المنطقة |
| حالة_العقار | string | ✅ | ❌ | الحالة (شاغر/مؤجر/صيانة) |
| الإيجار_التقديري | number | ❌ | ❌ | الإيجار التقديري |
| IsRented | boolean | ✅ | ❌ | هل العقار مؤجر؟ |
| المساحة | number | ✅ | ❌ | المساحة بالمتر المربع |
| الطابق | string | ❌ | ❌ | رقم الطابق |
| عدد_الغرف | string | ❌ | ❌ | عدد الغرف |
| نوع_التاثيث | string | ❌ | ❌ | نوع التأثيث |
| ملاحظات | string | ❌ | ❌ | ملاحظات إضافية |

**العلاقات:**
- `رقم_المالك` → `الأشخاص_tbl.رقم_الشخص` (Many-to-One)

**الفهارس:**
- `PK_PropertyID`: المفتاح الأساسي على `رقم_العقار`
- `IX_PropertyCode`: فهرس فريد على `الكود_الداخلي`
- `IX_FK_Property_Owner`: فهرس على `رقم_المالك`
- `IX_PropertyStatus`: فهرس على `حالة_العقار`
- `IX_PropertyType`: فهرس على `النوع`

**قواعد العمل:**
- لا يمكن حذف مالك له عقارات
- يجب أن يكون `IsRented = true` إذا كان هناك عقد نشط
- يجب أن يكون `IsRented = false` إذا لم يكن هناك عقد نشط

---

### 4️⃣ جدول العقود (العقود_tbl)

**الغرض:** تخزين معلومات عقود الإيجار

| الحقل | النوع | مطلوب | فريد | الوصف |
|------|------|------|------|-------|
| رقم_العقد | string | ✅ | ✅ PK | المفتاح الأساسي (CNT-timestamp) |
| رقم_العقار | string | ✅ | ❌ FK | مرجع لجدول العقارات |
| رقم_المستاجر | string | ✅ | ❌ FK | مرجع لجدول الأشخاص (العلاقة الوحيدة) |
| رقم_الكفيل | string | ❌ | ❌ | رقم الكفيل (اختياري - ليس علاقة رسمية) |
| تاريخ_البداية | string | ✅ | ❌ | تاريخ بداية العقد |
| تاريخ_النهاية | string | ✅ | ❌ | تاريخ نهاية العقد |
| مدة_العقد_بالاشهر | number | ✅ | ❌ | مدة العقد بالأشهر |
| القيمة_السنوية | number | ✅ | ❌ | القيمة السنوية للعقد |
| تكرار_الدفع | number | ✅ | ❌ | تكرار الدفع (شهري/ربع سنوي...) |
| طريقة_الدفع | string | ✅ | ❌ | طريقة الدفع (نقدي/شيك/تحويل) |
| قيمة_التأمين | number | ❌ | ❌ | قيمة التأمين |
| يوجد_دفعة_اولى | boolean | ❌ | ❌ | هل يوجد دفعة أولى؟ |
| قيمة_الدفعة_الاولى | number | ❌ | ❌ | قيمة الدفعة الأولى |
| حالة_العقد | string | ✅ | ❌ | الحالة (نشط/منتهي/ملغي) |
| isArchived | boolean | ✅ | ❌ | هل العقد مؤرشف؟ |

**العلاقات:**
- `رقم_العقار` → `العقارات_tbl.رقم_العقار` (Many-to-One)
- `رقم_المستاجر` → `الأشخاص_tbl.رقم_الشخص` (Many-to-One)

**⚠️ ملاحظة مهمة:**
- ✅ العلاقة الوحيدة المباشرة بين الأشخاص والعقود هي عبر `رقم_المستاجر` فقط
- ✅ حقل `رقم_الكفيل` موجود في العقد لكنه ليس علاقة رسمية (اختياري)
- ✅ المالك لا يرتبط مباشرة بالعقد، بل يُجلب من العقار: العقد → العقار → المالك

**الفهارس:**
- `PK_ContractID`: المفتاح الأساسي على `رقم_العقد`
- `IX_FK_Contract_Property`: فهرس على `رقم_العقار`
- `IX_FK_Contract_Tenant`: فهرس على `رقم_المستاجر`
- `IX_Contract_Status`: فهرس على `حالة_العقد`

**قواعد العمل:**
- لا يمكن أن يكون للعقار أكثر من عقد نشط واحد
- `تاريخ_النهاية` يجب أن يكون بعد `تاريخ_البداية`
- عند إنشاء عقد، يتم إنشاء الكمبيالات تلقائياً
- لا يمكن حذف مستأجر له عقود

---

### 5️⃣ جدول الكمبيالات (الكمبيالات_tbl)

**الغرض:** تخزين معلومات الكمبيالات (الأقساط)

| الحقل | النوع | مطلوب | فريد | الوصف |
|------|------|------|------|-------|
| رقم_الكمبيالة | string | ✅ | ✅ PK | المفتاح الأساسي |
| رقم_العقد | string | ✅ | ❌ FK | مرجع لجدول العقود |
| تاريخ_استحقاق | string | ✅ | ❌ | تاريخ الاستحقاق |
| القيمة | number | ✅ | ❌ | قيمة الكمبيالة |
| القيمة_المتبقية | number | ❌ | ❌ | المبلغ المتبقي (للدفعات الجزئية) |
| حالة_الكمبيالة | string | ✅ | ❌ | الحالة (مدفوع/غير مدفوع/متأخر) |
| تاريخ_الدفع | string | ❌ | ❌ | تاريخ الدفع الفعلي |
| نوع_الكمبيالة | string | ✅ | ❌ | النوع (إيجار/تأمين/دفعة أولى) |
| ترتيب_الكمبيالة | number | ❌ | ❌ | ترتيب الكمبيالة |
| سجل_الدفعات | array | ❌ | ❌ | سجل الدفعات الجزئية |
| ملاحظات | string | ❌ | ❌ | ملاحظات إضافية |

**العلاقات:**
- `رقم_العقد` → `العقود_tbl.رقم_العقد` (Many-to-One)

**الفهارس:**
- `PK_InstallmentID`: المفتاح الأساسي على `رقم_الكمبيالة`
- `IX_FK_Installment_Contract`: فهرس على `رقم_العقد`
- `IX_Installment_Status`: فهرس على `حالة_الكمبيالة`
- `IX_Installment_DueDate`: فهرس على `تاريخ_استحقاق`

**قواعد العمل:**
- يتم إنشاء الكمبيالات تلقائياً عند إنشاء العقد
- `القيمة_المتبقية` = `القيمة` في البداية
- عند الدفع الكامل: `حالة_الكمبيالة` = "مدفوع"
- عند الدفع الجزئي: `القيمة_المتبقية` تنقص

## 🔗 مخطط العلاقات

```
الأشخاص_tbl (1) ──────< (N) شخص_دور_tbl
     │
     ├──< (N) العقارات_tbl (رقم_المالك)
     │
     └──< (N) العقود_tbl (رقم_المستاجر) ← العلاقة الوحيدة المباشرة

العقارات_tbl (1) ──────< (N) العقود_tbl (رقم_العقار)

العقود_tbl (1) ──────< (N) الكمبيالات_tbl (رقم_العقد)
     │
     └──< (1) العمولات_tbl (رقم_العقد)
```

**⚠️ ملاحظة مهمة:**
- ✅ **العلاقة الوحيدة المباشرة بين الأشخاص والعقود هي `رقم_المستاجر` فقط**
- ✅ **المالك لا يرتبط مباشرة بالعقد**
- ✅ المالك يُجلب من العقار: `العقد → العقار → المالك`
- ✅ حقل `رقم_الكفيل` موجود في العقد لكنه **ليس علاقة رسمية** (اختياري)
- ✅ جميع الأدوار (مالك/مستأجر/كفيل) تُدار عبر `شخص_دور_tbl`

## ✅ قواعد سلامة البيانات

### 1. المفاتيح الأساسية (Primary Keys)
- ✅ يجب أن تكون فريدة
- ✅ لا يمكن أن تكون NULL
- ✅ يتم إنشاؤها تلقائياً باستخدام timestamp

### 2. القيود الفريدة (Unique Constraints)
- ✅ `الرقم_الوطني` في جدول الأشخاص
- ✅ `الكود_الداخلي` في جدول العقارات
- ✅ `اسم_المستخدم` في جدول المستخدمين

### 3. المفاتيح الخارجية (Foreign Keys)
- ✅ يجب أن تشير إلى سجل موجود
- ✅ عند الحذف: يتم التحقق من عدم وجود سجلات مرتبطة
- ✅ عند التحديث: يتم تحديث الفهارس تلقائياً

### 4. قواعد العمل (Business Rules)
- ✅ العقار المؤجر يجب أن يكون له عقد نشط
- ✅ العقار الشاغر لا يجب أن يكون له عقد نشط
- ✅ لا يمكن أن يكون للعقار أكثر من عقد نشط
- ✅ تاريخ النهاية يجب أن يكون بعد تاريخ البداية
