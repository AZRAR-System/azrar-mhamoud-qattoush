## 🎯 نظرة عامة

نظام المدفوعات والكمبيالات هو جزء أساسي من نظام إدارة العقارات AZRAR. يدير:

- ✅ **الكمبيالات (Installments)**: الدفعات الدورية المرتبطة بالعقود
- ✅ **العمولات (Commissions)**: عمولات الوسطاء الداخليين والخارجيين
- ✅ **السجلات المالية**: تتبع كامل لجميع المعاملات
- ✅ **التقارير**: تقارير مفصلة عن الإيرادات والمتأخرات
- ✅ **التنبيهات**: إشعارات تلقائية للدفعات المستحقة

---

## 🏗️ هيكل النظام

### 1️⃣ قاعدة البيانات

#### جدول الكمبيالات (`الكمبيالات_tbl`)
```typescript
interface الكمبيالات_tbl {
  رقم_الكمبيالة: string;        // PK: INST-YYYYMMDD-XXXX
  رقم_العقد: string;             // FK → العقود_tbl
  رقم_القسط: number;             // رقم الدفعة (1, 2, 3, ...)
  المبلغ: number;                 // قيمة الدفعة
  تاريخ_استحقاق: string;         // تاريخ الاستحقاق
  تاريخ_الدفع?: string;          // تاريخ الدفع الفعلي
  حالة_الكمبيالة: 'مدفوع' | 'غير مدفوع' | 'متأخر' | 'ملغي';
  المبلغ_المدفوع?: number;       // للدفعات الجزئية
  ملاحظات?: string;
  createdBy: string;
  createdAt: string;
  modifiedBy?: string;
  modifiedAt?: string;
}
```

#### جدول العمولات (`العمولات_tbl`)
```typescript
interface العمولات_tbl {
  رقم_العمولة: string;           // PK: COM-YYYYMMDD-XXXX
  رقم_العقد: string;             // FK → العقود_tbl
  رقم_الوسيط: string;            // FK → الأشخاص_tbl
  نوع_العمولة: 'نسبة مئوية' | 'مبلغ ثابت';
  قيمة_العمولة: number;         // النسبة أو المبلغ
  المبلغ_المحسوب: number;        // المبلغ النهائي
  حالة_الدفع: 'مدفوع' | 'غير مدفوع';
  تاريخ_الدفع?: string;
  ملاحظات?: string;
}
```

## ✨ الميزات الرئيسية

### 1️⃣ **إدارة الكمبيالات**

#### إنشاء تلقائي للكمبيالات
عند إنشاء/تجديد عقد، يتم إنشاء الكمبيالات تلقائيًا بناءً على:
- قيمة الإيجار
- نوع الدفع (شهري، ربع سنوي، نصف سنوي، سنوي)
- تاريخ البداية والنهاية

**مثال**:
```typescript
const contract = {
  قيمة_الإيجار: 12000,
  نوع_الدفع: 'شهري',
  تاريخ_البداية: '2026-01-01',
  تاريخ_النهاية: '2026-12-31'
};
// ينتج: 12 كمبيالة (1000 دينار لكل شهر)
```

#### تسجيل الدفعات
- **دفعة كاملة**: يُسجّل المبلغ الكامل ويُحدّث الحالة إلى "مدفوع"
- **دفعة جزئية**: يُسجّل المبلغ المدفوع ويبقى الرصيد المتبقي
- **دفعات متعددة**: يمكن تسجيل عدة دفعات جزئية لنفس الكمبيالة

#### عكس السداد (Reversal)
- ⚠️ **صلاحية محدودة**: SuperAdmin فقط
- 📝 **سجل تدقيق**: يُسجّل السبب والمستخدم
- 🔒 **آمن**: لا يحذف السجلات، يُرجع الحالة فقط

---

### 2️⃣ **إدارة العمولات**

#### أنواع العمولات
1. **عمولة داخلية**: للوسطاء الموظفين
2. **عمولة خارجية**: للوسطاء المستقلين

#### حساب تلقائي
```typescript
// نسبة مئوية
const commission = {
  نوع_العمولة: 'نسبة مئوية',
  قيمة_العمولة: 5,
  قيمة_الإيجار: 12000
};
// النتيجة: 12000 × 5% = 600 دينار

// مبلغ ثابت
const commission = {
  نوع_العمولة: 'مبلغ ثابت',
  قيمة_العمولة: 500
};
// النتيجة: 500 دينار
```

## 🚀 دليل الاستخدام السريع

### 1️⃣ **عرض الكمبيالات**
```
الصفحة: الكمبيالات (Installments)
المسار: /#/installments
```

**الفلترة**:
- حسب الحالة (الكل / مدفوع / غير مدفوع / متأخر)
- حسب المستأجر
- حسب العقار
- حسب التاريخ

---

### 2️⃣ **تسجيل دفعة**

**الخطوات**:
1. افتح صفحة الكمبيالات
2. ابحث عن الكمبيالة المطلوبة
3. اضغط **"تسجيل دفعة"**
4. أدخل:
   - المبلغ المدفوع
   - تاريخ الدفع
   - طريقة الدفع (نقدي / شيك / تحويل)
   - ملاحظات (اختياري)
5. اضغط **"حفظ"**

**النتيجة**:
- ✅ يُحدّث حالة الكمبيالة
- ✅ يُسجّل في سجل العمليات (Logs)
- ✅ يُحدّث التقارير المالية

---

### 3️⃣ **عكس دفعة (Reversal)**

⚠️ **متاح فقط لـ SuperAdmin**

**الخطوات**:
1. افتح صفحة الكمبيالات
2. ابحث عن الكمبيالة المدفوعة
3. اضغط **"عكس السداد"**
4. أدخل **السبب** (إلزامي)
5. اضغط **"تأكيد"**

**النتيجة**:
- ✅ تُرجع حالة الكمبيالة إلى "غير مدفوع"
- ✅ يُسجّل في سجل التدقيق (Audit Log)
- 📝 السجل السابق يبقى (لا يُحذف)

---

### 4️⃣ **إضافة عمولة**

**الخطوات**:
1. افتح صفحة العقود
2. افتح تفاصيل العقد
3. انتقل لتبويب **"العمولات"**
4. اضغط **"إضافة عمولة"**
5. أدخل:
   - الوسيط (من قائمة الوسطاء)
   - نوع العمولة (نسبة / مبلغ ثابت)
   - القيمة
6. اضغط **"حفظ"**

**النتيجة**:
- ✅ يُحسب المبلغ تلقائيًا
- ✅ يُضاف لجدول العمولات

## 🐛 المشاكل المعروفة والحلول

### ⚠️ مشكلة: الكمبيالات لا تُنشأ تلقائيًا

**السبب**: خطأ في حساب عدد الأقساط أو التواريخ

**الحل**:
```typescript
// تحقق من بيانات العقد
const contract = DbService.getContractById(contractId);
console.log({
  قيمة_الإيجار: contract.قيمة_الإيجار,
  نوع_الدفع: contract.نوع_الدفع,
  تاريخ_البداية: contract.تاريخ_البداية,
  تاريخ_النهاية: contract.تاريخ_النهاية
});

// إعادة إنشاء الكمبيالات
DbService.generateInstallments(contractId);
```

---

### ⚠️ مشكلة: الدفعات الجزئية لا تُحسب بشكل صحيح

**السبب**: عدم تحديث `المبلغ_المدفوع`

**الحل**:
```typescript
// التحقق من المبلغ المدفوع
const installment = DbService.getInstallmentById(installmentId);
const paid = installment.المبلغ_المدفوع || 0;
const remaining = installment.المبلغ - paid;

console.log({
  المبلغ_الكلي: installment.المبلغ,
  المبلغ_المدفوع: paid,
  المبلغ_المتبقي: remaining
});
```

---

### ⚠️ مشكلة: العمولات لا تُحسب تلقائيًا

**السبب**: نوع العمولة خاطئ أو قيمة الإيجار غير موجودة

**الحل**:
```typescript
// إعادة حساب العمولة
const contract = DbService.getContractById(contractId);
const commission = DbService.getCommissionById(commissionId);

let calculatedAmount = 0;
if (commission.نوع_العمولة === 'نسبة مئوية') {
  calculatedAmount = (contract.قيمة_الإيجار * commission.قيمة_العمولة) / 100;
} else {
  calculatedAmount = commission.قيمة_العمولة;
}

DbService.updateCommission(commissionId, { المبلغ_المحسوب: calculatedAmount });
```

## 🔒 الأمان والصلاحيات

### نظام RBAC

#### الصلاحيات المطلوبة:

| العملية | الصلاحية المطلوبة | الأدوار |
|---------|-------------------|---------|
| عرض الكمبيالات | `INSTALLMENT_VIEW` | الكل |
| تسجيل دفعة | `INSTALLMENT_PAY` | Manager, Admin, SuperAdmin |
| عكس دفعة | `INSTALLMENT_REVERSE` | **SuperAdmin فقط** |
| إضافة عمولة | `COMMISSION_ADD` | Manager, Admin, SuperAdmin |
| دفع عمولة | `COMMISSION_PAY` | Manager, Admin, SuperAdmin |

---

### مثال: التحقق من الصلاحية

```typescript
import { can } from '@/utils/permissions';
import { useAuth } from '@/context/AuthContext';

const { user } = useAuth();

// عرض زر "عكس السداد" فقط للسوبر أدمن
{can(user?.role, 'INSTALLMENT_REVERSE') && (
  <button onClick={handleReverse}>
    عكس السداد
  </button>
)}
```

## 📊 الإحصائيات

### حجم النظام:
- ✅ **الكمبيالات**: جدول واحد (~15 حقل)

- ⚡ **سرعة الكتابة**: <50ms
- 🔄 **التزامن**: دعم كامل (localStorage + SQLite)
