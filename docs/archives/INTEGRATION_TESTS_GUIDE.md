# 🧪 دليل الاختبارات التسلسلية

## 📋 نظرة عامة

تم إنشاء مجموعة اختبارات شاملة تختبر النظام بشكل متسلسل:

```
شخص (مالك) → عقار → عقد إيجار → كمبيالات → عمولات
```

كل اختبار يعتمد على نتائج الاختبار السابق، مما يضمن تكامل النظام.

## 🚀 كيفية التشغيل

### الطريقة 1️⃣: عبر واجهة المستخدم

1. اذهب إلى صفحة الاختبارات:
   ```
   /integration-tests
   ```

2. اضغط على "تشغيل الاختبارات"

3. راقب النتائج في الوقت الفعلي

4. حمّل التقرير كملف JSON

### الطريقة 2️⃣: عبر DevTools (المتصفح)

في `Console` اكتب:
```javascript
// تشغيل جميع الاختبارات
await runTests();

// أو استخدم الفئة مباشرة
const suite = new IntegrationTestSuite();
await suite.runAllTests();
```

### الطريقة 3️⃣: من الكود

```typescript
import { IntegrationTestSuite } from './services/integrationTests';

async function runTests() {
  const suite = new IntegrationTestSuite();
  const results = await suite.runAllTests();
  console.log(results);
}

runTests();
```

## 📊 قراءة النتائج

### حالات الاختبار

| الحالة | الرمز | المعنى |
|--------|------|--------|
| نجح | ✅ | الاختبار نجح |
| فشل | ❌ | الاختبار فشل |
| تم تخطيه | ⏭️ | الاختبار تم تخطيه (الاختبار السابق فشل) |

### ملخص النتائج

```
✅ نجح:        7/7
❌ فشل:        0/7
⏭️ تم تخطيه:  0/7
⏱️ المدة الكلية: 245.32ms
```

### البيانات التفصيلية

كل اختبار يعيد:
- `testName`: اسم الاختبار
- `status`: نتيجة الاختبار
- `message`: رسالة توضيحية
- `data`: البيانات المفصلة
- `duration`: المدة الزمنية

## 🔍 استكشاف الأخطاء

### المشكلة: الاختبار الأول فشل

**السبب:** مشكلة في إضافة الشخص

**الحل:**
1. تحقق من أن `DbService.addPerson()` يعمل
2. تحقق من البيانات المدخلة
3. تأكد من أن localStorage متاح

### المشكلة: الاختبارات متوقفة في المنتصف

**السبب:** اختبار سابق فشل

**الحل:**
1. اقرأ رسالة الخطأ بعناية
2. صحح الخطأ
3. شغّل الاختبارات من جديد

### المشكلة: "أرقام الكمبيالات" غير صحيحة

**السبب:** مشكلة في حساب عدد الكمبيالات

**الحل:**
```javascript
// الصيغة الصحيحة:
// إذا كان التكرار 12 شهري:
// الكمبيالات = 1 (تأمين) + 12 (إيجار) = 13 كمبيالة
```
