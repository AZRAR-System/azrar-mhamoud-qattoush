# تفعيل AZRAR بملف موقّع (Device-bound License)

## الفكرة
- يتم إنشاء ملف تفعيل يحتوي على `deviceId` (بصمة الجهاز) + بيانات (اختياري: اسم العميل/تاريخ الانتهاء).
- يتم توقيع الملف بمفتاح خاص (Private Key) عند الدعم.
- داخل برنامج AZRAR يتم التحقق بالمفتاح العام (Public Key) فقط.

هذا يمنع تعديل ملف التفعيل (tampering) ويضمن أنه يعمل فقط على الجهاز المحدد.

## البيع كاشتراكات + عدد أجهزة (Offline-First)

بما أن كثيرًا من العملاء لا يملكون إنترنت، أفضل نموذج عملي هو:

- **ملف تفعيل مُوقّع لكل جهاز** (Seat = جهاز واحد).
- **مدة صلاحية داخل الملف** (`expiresAt`) حسب نوع الاشتراك:
  - شهري / 3 أشهر / نصف سنوي / سنوي

بهذا الشكل:
- لا يمكن تزوير ملف التفعيل بدون المفتاح الخاص.
- لا يمكن استخدام نفس الملف على أكثر من جهاز (لأنه مرتبط بـ `deviceId`).
- التحكم “بعد البيع” يتم عبر **عدم تجديد الاشتراك** (إرسال ملف جديد) بدل الاعتماد على إنترنت دائم.

ملاحظة مهمة: في نظام Offline بالكامل لا يوجد إلغاء فوري (revocation) على أجهزة لا تتصل إطلاقًا، لكن مدة الترخيص القصيرة + التجديد تمنحك سيطرة قوية.

### كيف يتم تفعيل عدة أجهزة لنفس العميل؟
1) كل جهاز يعرض `deviceId` من شاشة التفعيل.
2) الدعم يولّد **ملفًا منفصلًا لكل جهاز** بنفس بيانات العميل/المدة.
3) يتم تسليم كل ملف للعميل وتثبيته على جهازه المقابل.

> ملاحظة: لا يوجد نظام «لا ينكسر أبداً» ضد التعديل العميق/الهندسة العكسية، لكن التوقيع الرقمي + ربط البصمة هو أقوى حل عملي ومستقر.

---

## 1) توليد مفاتيح التفعيل
الخيار الأسهل (بدون أوامر):

- افتح برنامج مولّد التفعيل **AZRAR-LicenseGen**
- اضغط زر **توليد مفتاح جديد**
- احفظ الملف

سيتم إنشاء ملفين في نفس المجلد:
- `azrar-license-private.key.json` (سري جداً)
- `azrar-license-public.key.json`

بديل (سطر أوامر):
- `npm run license:keygen`

---

## 2) إعداد AZRAR للتحقق بالمفتاح العام
عند بناء برنامج AZRAR (العميل):

- ضع قيمة `publicKeyB64` في متغير البيئة:
  - `VITE_AZRAR_LICENSE_PUBLIC_KEY=<base64>`

ثم ابنِ نسخة الديسكتوب كالمعتاد.

---

## 3) إنشاء ملف تفعيل للعميل (برنامج مولّد التفعيل)
- تشغيل مولّد التفعيل (للتجربة):
  - `npm run licensegen:dev`

إذا لم تظهر نافذة مولّد التفعيل أو أغلق مباشرة، شغّل الوضع التشخيصي:
- `npm run licensegen:dev:verbose`

وسيتم حفظ سجل التشغيل في:
- `tmp/licensegen-dev.log`

- إنشاء Installer لمولّد التفعيل:
  - `npm run licensegen:dist`

وسيتم إخراج الـ Setup في:
- `release_licensegen_build/AZRAR-LicenseGen-Setup-<version>.exe`

بديل (بدون واجهة) لإنشاء ملف تفعيل من سطر الأوامر:
- `npm run license:sign -- --privateKey secrets/azrar-license-private.key.json --deviceId <DEVICE_ID> --out tmp/license.json`

وللتأكد أن الملف صحيح وسيُقبل (التوقيع + ربط البصمة):
- `npm run license:verify:file -- tmp/license.json --deviceId <DEVICE_ID>`

الخطوات داخل مولّد التفعيل:
1) تحميل المفتاح الخاص من `azrar-license-private.key.json`
2) لصق `deviceId` من جهاز العميل
3) (اختياري) اسم العميل / تاريخ الانتهاء
4) إنشاء الملف وحفظه

---

## 4) تفعيل AZRAR عند العميل
في شاشة تسجيل الدخول:
- اضغط "تحميل ملف التفعيل" واختر ملف التفعيل.

إذا كانت البصمة صحيحة والتوقيع صحيح ولم تنته الصلاحية، سيصبح النظام مُفعّل.

ملاحظة أمنية:
- في نسخة الإنتاج، التفعيل المعتمد هو ملف تفعيل مُوقّع فقط. أي تعديل يدوي على التخزين المحلي لن يُفعّل النظام بدون توقيع صحيح.

## حماية إضافية لاشتراكات المدة (بدون إنترنت)

- النظام يحاول كشف **الرجوع الكبير بوقت الجهاز للخلف** لمنع التحايل على تاريخ الانتهاء.
- إذا واجه العميل رسالة عن الوقت، يتم ضبط التاريخ/الوقت الصحيح ثم إعادة تشغيل البرنامج.
