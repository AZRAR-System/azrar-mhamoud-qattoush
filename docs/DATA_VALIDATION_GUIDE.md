# ✅ دليل التحقق من صحة البيانات

## 📋 نظرة عامة

تم تطبيق نظام شامل للتحقق من صحة البيانات في جميع مستويات النظام لضمان:
- ✅ عدم تكرار المفاتيح الأساسية
- ✅ عدم تكرار القيود الفريدة
- ✅ سلامة المفاتيح الخارجية
- ✅ صحة قواعد العمل

---

## 🔍 مستويات التحقق

### 1️⃣ التحقق عند الإدخال (Input Validation)

يتم التحقق من البيانات **قبل** إضافتها إلى قاعدة البيانات.

#### إضافة شخص جديد:
```typescript
// في peopleService.ts
export const addPerson = (data, roles) => {
  // ✅ التحقق من صحة البيانات
  const validation = validateNewPerson(data);
  if (!validation.isValid) {
    return fail(validation.errors.join(', '));
  }
  // ... إضافة الشخص
}
```

**التحققات:**
- ✅ الاسم مطلوب ولا يمكن أن يكون فارغاً
- ✅ رقم الهاتف مطلوب ولا يمكن أن يكون فارغاً
- ⚠️ رقم الهاتف يجب أن يبدأ بـ 07 ويتكون من 10 أرقام
- ⚠️ الرقم الوطني يجب أن يتكون من 10 أرقام
- ❌ الرقم الوطني يجب أن يكون فريداً (لا تكرار)

#### إضافة عقار جديد:
```typescript
// في propertiesService.ts
export const addProperty = (data) => {
  // ✅ التحقق من صحة البيانات
  const validation = validateNewProperty(data);
  if (!validation.isValid) {
    return fail(validation.errors.join(', '));
  }
  // ... إضافة العقار
}
```

**التحققات:**
- ✅ الكود الداخلي مطلوب ولا يمكن أن يكون فارغاً
- ✅ المالك مطلوب ويجب أن يكون موجوداً
- ✅ نوع العقار مطلوب
- ✅ العنوان مطلوب
- ❌ الكود الداخلي يجب أن يكون فريداً (لا تكرار)

---

### 2️⃣ التحقق من المفاتيح الأساسية (Primary Keys)

```typescript
checkPrimaryKeyDuplicates()
```

**يتحقق من:**
- ✅ عدم تكرار `رقم_الشخص` في جدول الأشخاص
- ✅ عدم تكرار `رقم_العقار` في جدول العقارات
- ✅ عدم تكرار `رقم_العقد` في جدول العقود
- ✅ عدم تكرار `رقم_الكمبيالة` في جدول الكمبيالات

**مثال على الخطأ:**
```
❌ تكرار في رقم الشخص: P-1234567890
```

---

### 3️⃣ التحقق من القيود الفريدة (Unique Constraints)

```typescript
checkUniqueConstraints()
```

**يتحقق من:**
- ✅ عدم تكرار `الرقم_الوطني` في جدول الأشخاص
- ✅ عدم تكرار `الكود_الداخلي` في جدول العقارات
- ✅ عدم تكرار `اسم_المستخدم` في جدول المستخدمين

**مثال على الخطأ:**
```
❌ تكرار في الرقم الوطني: 1234567890
❌ تكرار في الكود الداخلي للعقار: A-101
```

---

### 4️⃣ التحقق من سلامة المفاتيح الخارجية (Foreign Key Integrity)

```typescript
checkForeignKeyIntegrity()
```

**يتحقق من:**
- ✅ كل عقار له مالك موجود في جدول الأشخاص
- ✅ كل عقد له عقار موجود في جدول العقارات
- ✅ كل عقد له مستأجر موجود في جدول الأشخاص
- ⚠️ كل عقد له كفيل (إن وجد) موجود في جدول الأشخاص
- ✅ كل كمبيالة لها عقد موجود في جدول العقود
- ✅ كل دور له شخص موجود في جدول الأشخاص

**مثال على الخطأ:**
```
❌ العقار A-101: المالك P-999 غير موجود
❌ العقد CNT-001: العقار PROP-999 غير موجود
⚠️ العقد CNT-002: الكفيل P-888 غير موجود
```

---

### 5️⃣ التحقق من قواعد العمل (Business Logic)

```typescript
checkBusinessLogic()
```

**يتحقق من:**
- ⚠️ العقار المحدد كمؤجر يجب أن يكون له عقد نشط
- ⚠️ العقار المحدد كشاغر لا يجب أن يكون له عقد نشط
- ❌ العقار لا يجب أن يكون له أكثر من عقد نشط واحد
- ❌ تاريخ نهاية العقد يجب أن يكون بعد تاريخ البداية
- ⚠️ مدة العقد يجب أن تتطابق مع الفرق بين التواريخ

**مثال على الخطأ:**
```
⚠️ العقار A-101: محدد كمؤجر لكن لا يوجد عقد نشط
❌ العقار B-205: يوجد أكثر من عقد نشط (2)
❌ العقد CNT-003: تاريخ النهاية قبل تاريخ البداية
```

## 🛠️ كيفية استخدام التحقق

### التحقق الشامل من كل شيء:

```typescript
import { validateAllData } from '@/services/dataValidation';

const result = validateAllData();

if (result.isValid) {
  console.log('✅ جميع البيانات صحيحة!');
} else {
  console.error('❌ أخطاء:', result.errors);
  console.warn('⚠️ تحذيرات:', result.warnings);
}
```

### التحقق قبل إضافة شخص:

```typescript
import { validateNewPerson } from '@/services/dataValidation';

const validation = validateNewPerson({
  الاسم: 'أحمد محمد',
  رقم_الهاتف: '0791234567',
  الرقم_الوطني: '1234567890'
});

if (!validation.isValid) {
  alert(validation.errors.join('\n'));
}
```

### التحقق قبل إضافة عقار:

```typescript
import { validateNewProperty } from '@/services/dataValidation';

const validation = validateNewProperty({
  الكود_الداخلي: 'A-101',
  رقم_المالك: 'P-1234567890',
  النوع: 'شقة',
  العنوان: 'عمان - الدوار الرابع'
});

if (!validation.isValid) {
  alert(validation.errors.join('\n'));
}
```

## 📊 أنواع الرسائل

### ❌ أخطاء (Errors)
- **تمنع** العملية من الاستمرار
- يجب إصلاحها قبل المتابعة
- مثال: تكرار المفتاح الأساسي، مرجع خارجي غير موجود

### ⚠️ تحذيرات (Warnings)
- **لا تمنع** العملية من الاستمرار
- يُنصح بإصلاحها لكنها ليست إجبارية
- مثال: صيغة رقم الهاتف غير صحيحة، عدم تطابق التواريخ

## 🔧 الدوال المتاحة

| الدالة | الوصف | الإرجاع |
|-------|-------|---------|
| `validateAllData()` | التحقق الشامل من كل شيء | `ValidationResult` |
| `checkPrimaryKeyDuplicates()` | التحقق من تكرار المفاتيح الأساسية | `ValidationResult` |
| `checkUniqueConstraints()` | التحقق من القيود الفريدة | `ValidationResult` |
| `checkForeignKeyIntegrity()` | التحقق من سلامة المفاتيح الخارجية | `ValidationResult` |
| `checkBusinessLogic()` | التحقق من قواعد العمل | `ValidationResult` |
| `validateNewPerson(data)` | التحقق قبل إضافة شخص | `ValidationResult` |
| `validateNewProperty(data)` | التحقق قبل إضافة عقار | `ValidationResult` |

## 📝 نوع ValidationResult

```typescript
interface ValidationResult {
  isValid: boolean;      // هل البيانات صحيحة؟
  errors: string[];      // قائمة الأخطاء
  warnings: string[];    // قائمة التحذيرات
}
```

## ✅ أفضل الممارسات

1. **دائماً** استخدم التحقق قبل إضافة بيانات جديدة
2. **لا تتجاهل** الأخطاء - يجب إصلاحها
3. **انتبه** للتحذيرات - قد تسبب مشاكل لاحقاً
4. **استخدم** `validateAllData()` بشكل دوري للتحقق من سلامة البيانات
5. **اعرض** رسائل الخطأ للمستخدم بشكل واضح
