## معلومات الإصدارات

### Semantic Versioning

نستخدم نظام الترقيم الدلالي (SemVer):
- **MAJOR (الرئيسي):** تغييرات غير متوافقة مع الإصدارات السابقة
- **MINOR (الفرعي):** إضافة مميزات متوافقة مع السابق
- **PATCH (التصحيح):** إصلاحات متوافقة

مثال: `2.0.109`
- `2` = الإصدار الرئيسي
- `0` = الإصدار الفرعي
- `109` = رقم التصحيح

## المساهمة في السجل

عند إضافة تغييرات جديدة:

1. أضف التغيير في قسم `[غير المُصدَر]`
2. استخدم الأنواع المحددة أعلاه
3. اكتب وصفاً واضحاً ومختصراً
4. أضف رقم PR إن وجد: `(#123)`

مثال:
```markdown
### الإضافات
- إضافة نظام الدفع الإلكتروني (#456)
```
