## 🚨 الإبلاغ عن ثغرة أمنية

### لا تبلّغ علناً!

إذا اكتشفت ثغرة أمنية، **لا تفتح Issue عامة**. بدلاً من ذلك:

### طرق الإبلاغ الآمنة:

1. **GitHub Security Advisory (مفضل):**
   - اذهب إلى: `Security` → `Report a vulnerability`
   - املأ النموذج بالتفاصيل

2. **قناة داخلية (لأعضاء المشروع):** استخدم قناة الفريق الخاصة أو ارفع Security Advisory ثم شارك الرابط داخلياً.

### ما يجب تضمينه في التقرير:

- **الوصف:** وصف مفصل للثغرة
- **التأثير:** ما الذي يمكن أن يحدث؟
- **الخطوات:** كيف يمكن إعادة إنتاج المشكلة؟
- **الإصدار:** أي إصدار من AZRAR متأثر؟
- **البيئة:** نظام التشغيل، إعدادات، إلخ
- **POC:** إثبات المفهوم (إن أمكن)
- **معلومات الاتصال:** بريدك الإلكتروني

### ماذا تتوقع:

| الوقت | الإجراء |
|-------|---------|
| 24 ساعة | تأكيد استلام التقرير |
| 3 أيام | تقييم أولي |
| 7 أيام | خطة الإصلاح |
| 30 يوم | إصدار التحديث (للثغرات الحرجة) |

### برنامج المكافآت

حالياً لا يوجد برنامج مكافآت عام موثّق لهذا المشروع.

## 🔒 ممارسات الأمان المطبقة

### 1. أمان الكود

```typescript
// ✅ جيد: استخدام Prepared Statements
const result = await db.prepare('SELECT * FROM users WHERE id = ?').get(userId);

// ❌ سيء: SQL Injection
const result = await db.exec(`SELECT * FROM users WHERE id = ${userId}`);
```

- ✅ **Parameterized Queries:** منع SQL Injection
- ✅ **Input Validation:** تحقق من جميع المدخلات
- ✅ **Output Encoding:** منع XSS
- ✅ **Content Security Policy:** حماية من السكريبتات الخبيثة

### 2. أمان Electron

```javascript
// في main.js
const mainWindow = new BrowserWindow({
  webPreferences: {
    nodeIntegration: false,        // ✅ معطّل
    contextIsolation: true,        // ✅ مفعّل
    sandbox: true,                 // ✅ مفعّل
    enableRemoteModule: false,     // ✅ معطّل
    webSecurity: true              // ✅ مفعّل
  }
});
```

- ✅ **Context Isolation:** فصل سياقات التنفيذ
- ✅ **Sandbox:** بيئة محدودة للصفحات
- ✅ **No Node Integration:** منع الوصول المباشر لـ Node.js
- ✅ **Preload Scripts:** جسر آمن بين Renderer و Main

### 3. أمان البيانات

```javascript
// تشفير قاعدة البيانات
const Database = require('better-sqlite3');
const db = new Database('azrar.db', {
  // التشفير مفعّل
});

// تشفير كلمات المرور
const bcrypt = require('bcrypt');
const hashedPassword = await bcrypt.hash(password, 10);
```

- ✅ **التشفير:** جميع البيانات الحساسة مشفرة
- ✅ **Hashing:** كلمات المرور مع bcrypt
- ✅ **Secure Storage:** حفظ آمن للبيانات الحساسة
- ✅ **Session Management:** إدارة جلسات آمنة

### 4. التحديثات الآمنة

```javascript
// في main.js
autoUpdater.setFeedURL({
  provider: 'github',
   owner: 'AZRAR-System',
   repo: 'azrar-mhamoud-qattoush'
});

// التحقق من التوقيع
autoUpdater.verifyUpdateCodeSignature = true;
```

- ✅ **Code Signing:** جميع التحديثات موقعة رقمياً
- ✅ **HTTPS Only:** تحميل عبر HTTPS فقط
- ✅ **Integrity Check:** التحقق من SHA512
- ✅ **Rollback:** إمكانية العودة للإصدار السابق

## 🛡️ الثغرات المعروفة والمخففة

### 1. أمثلة على أنواع مشاكل يتم التعامل معها

> ملاحظة: هذا القسم وصفي/تعليمي، وليس سجل CVE رسمي.

- Path Traversal
- XSS
- SQL Injection
- Misconfigured Electron webPreferences

### 2. قيود معروفة

#### أ. قاعدة البيانات المحلية

- **المشكلة:** SQLite غير مشفر افتراضياً
- **التخفيف:** استخدام SQLCipher في الإصدارات القادمة
- **الحل المؤقت:** حماية على مستوى نظام الملفات

#### ب. تخزين بيانات الاعتماد

- **المشكلة:** تخزين في ملف .env
- **التخفيف:** استخدام Keytar لنظام التشغيل
- **الخطة:** انتقال كامل في v2.1

## 🔧 إعدادات الأمان الموصى بها

### للمستخدمين:

1. **تفعيل جدار الحماية:**
   ```
   Settings → Security → Enable Firewall Protection
   ```

2. **استخدام كلمات مرور قوية:**
   - 12 حرف على الأقل
   - أحرف كبيرة وصغيرة
   - أرقام ورموز

3. **تفعيل النسخ الاحتياطي التلقائي:**
   ```
   Settings → Backup → Auto-backup (Daily)
   ```

4. **تحديث التطبيق دائماً:**
   - تفعيل التحديثات التلقائية
   - أو تحقق يدوياً أسبوعياً

### للمطورين:

1. **مراجعة الكود:**
   - استخدم ESLint مع قواعد الأمان
   - مراجعة الأقران لكل PR

2. **فحص التبعيات:**
   ```bash
   npm audit
   npm audit fix
   ```

3. **اختبارات الأمان:**
   ```bash
   npm run security:test
   npm run security:scan
   ```

4. **متغيرات البيئة:**
   ```bash
   # لا تحفظ أبداً في Git
   echo ".env" >> .gitignore
   echo "*.pfx" >> .gitignore
   ```
